# FactFeed 📰⚖️

**FactFeed** — это умный новостной агрегатор, который помогает отличать факты от мнений. Используя передовые методы NLP (обработки естественного языка), он собирает новости из различных RSS-источников и классифицирует каждое предложение как **Факт**, **Мнение** или **Смешанное утверждение**.

## Возможности

-   **🔍 Умный поиск**: Полнотекстовый поиск по статьям с фильтрацией по источнику и дате.
-   **📊 Фактометр**: Визуальный индикатор соотношения фактов и мнений для каждой статьи.
-   **🧠 NLP Анализ**: Использует трансформеры (DeBERTa) и spaCy для классификации предложений.
-   **⚡ Быстрый UI**: Server-side rendering на базе **FastAPI** и **HTMX** для мгновенной загрузки.
-   **🌍 Локализация**: Полная поддержка русского и английского языков.
-   **🔌 Публичный API**: RESTful API для доступа к размеченным данным.

## Управление проектом (Makefile)

В корне проекта находится `Makefile`, который автоматизирует все рутинные задачи. Вам не нужно запоминать длинные команды консоли.

### Основные команды

| Команда | Описание |
|---------|----------|
| `make install` | Установить зависимости (используя `uv`) |
| `make run` | Запустить сервер разработки локально (порт 8001, использует БД на 5433) |
| `make docker-up` | Запустить всё приложение в Docker (App + DB) |
| `make docker-down` | Остановить Docker контейнеры |
| `make test` | Запустить тесты |
| `make clean` | Очистить кэш и временные файлы |

### Работа с Базой Данных

| Команда | Описание |
|---------|----------|
| `make migrate` | Применить миграции (обновить схему БД) |
| `make revision` | Создать новую миграцию (интерактивный режим) |

### Локализация (i18n)

Система переводов построена на библиотеке `Babel`.

| Команда | Описание |
|---------|----------|
| `make i18n-extract` | Извлечь строки для перевода из кода и шаблонов в `.pot` файл |
| `make i18n-init` | Инициализировать новый язык (интерактивно) |
| `make i18n-update` | Обновить файлы переводов `.po` (если изменился исходный код) |
| `make i18n-compile` | Скомпилировать переводы в бинарные `.mo` файлы (нужно после правки перевода) |

## Быстрый старт

### Вариант 1: Docker (Рекомендуется)

Самый простой способ запустить проект, не устанавливая Python и зависимости локально.

1.  **Запуск**:
    ```bash
    make docker-up
    ```
    Это поднимет PostgreSQL, выполнит миграции и запустит веб-сервер.

2.  **Доступ**:
    Откройте [http://localhost:8001](http://localhost:8001) в браузере.

    *При первом запуске пайплайн сбора новостей начнет работать в фоне и наполнит базу статьями в течение нескольких минут.*

### Вариант 2: Локальная разработка

Если вы хотите вести разработку кода. Требуется Python 3.12+ и [uv](https://github.com/astral-sh/uv).

1.  **Установка зависимостей**:
    ```bash
    make install
    ```

2.  **Запуск базы данных** (требуется Docker только для Postgres):
    ```bash
    docker-compose up -d postgres
    ```

3.  **Применение миграций**:
    ```bash
    make migrate
    ```

4.  **Запуск сервера**:
    ```bash
    make run
    ```
    Приложение будет доступно по адресу [http://localhost:8001](http://localhost:8001).

    > **Важно**: При локальном запуске приложение подключается к базе данных на порту **5433** (маппинг Docker), чтобы избежать конфликтов с локальным Postgres на 5432. `make run` настраивает это автоматически. Если вы запускаете команды вручную, установите переменную:
    > `export DATABASE_URL=postgresql+asyncpg://factfeed:factfeed@localhost:5433/factfeed`

## API Документация

Интерактивная документация (Swagger UI) доступна по адресу `/docs`.

Ключевые эндпоинты:
-   `GET /api/v1/search`: Поиск статей с фильтрами.
-   `GET /api/v1/articles/{id}`: Детальная информация о статье и разметка предложений.
-   `GET /api/v1/sources`: Список доступных источников новостей.

## Стек технологий

-   **Backend**: Python 3.12, FastAPI, SQLAlchemy (Async), Pydantic.
-   **Database**: PostgreSQL 16.
-   **NLP**: HuggingFace Transformers, PyTorch, spaCy.
-   **Frontend**: Jinja2 Templates, HTMX, CSS Variables.
-   **DevOps**: Makefile, UV, Docker Compose.
