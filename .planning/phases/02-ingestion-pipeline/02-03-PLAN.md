**Create `factfeed/ingestion/persister.py`** with two async functions:

**`async def save_article(session: AsyncSession, article_data: dict) -> bool`:**
- Use PostgreSQL `INSERT ... ON CONFLICT DO NOTHING` via `sqlalchemy.dialects.postgresql.insert`
- Build insert statement: `pg_insert(Article).values(**article_data)`
- Add `.on_conflict_do_nothing(index_elements=["url_hash"])`
- Execute and commit. Return `result.rowcount == 1` (True if inserted, False if duplicate skipped)

**`async def seed_sources(session: AsyncSession, sources: list[dict]) -> None`:**
- For each source in the list, use `pg_insert(Source).values(name=source["name"], feed_url=source["feed_url"]).on_conflict_do_nothing(index_elements=["feed_url"])`
- Execute all inserts, commit once
- Log number of sources seeded at INFO level

**Create `factfeed/ingestion/runner.py`:**

**Module-level state:**
- `_failure_counts: dict[str, int] = {}` — tracks consecutive failures per source name
- `log = structlog.get_logger()`

**`async def run_ingestion_cycle(session_factory, http_client: httpx.AsyncClient) -> dict`:**
- Generate a cycle_id: `str(uuid.uuid4())[:8]`
- Log `ingestion_cycle_start` at INFO with cycle_id and source_count
- Query all sources from DB: `await session.execute(select(Source))` to get the list of sources with their IDs
- Use `asyncio.gather(*[fetch_rss_feed(source_dict, http_client) for source_dict in sources_list], return_exceptions=True)` to fetch all feeds concurrently
- For each source + feed result:
  - If result is an Exception: call `_log_source_error(source_name, str(result))`, continue to next source
  - Otherwise call `_process_source_entries(source, feed_result, http_client, session_factory)` and reset failure count on success
- Log `ingestion_cycle_end` at INFO with cycle_id and aggregate stats
- Return aggregate stats dict

**`async def _process_source_entries(source, feed, http_client, session_factory) -> dict`:**
- Initialize stats: `{"found": len(feed.entries), "inserted": 0, "skipped": 0, "errors": 0}`
- For each entry in `feed.entries`:
  - Extract URL from `entry.get("link", "")` — skip if empty
  - Compute `url_hash = compute_url_hash(url)`
  - Check `article_exists(url_hash, session)` — if True, increment skipped, log at DEBUG, continue
  - Check `can_fetch(url, settings.user_agent, http_client)` — if False, increment skipped, log at DEBUG, continue
  - Fetch article page: `html_bytes = await fetch_article_page(url, http_client)` — if None, use partial fallback
  - Extract article: `extracted = extract_article(html_bytes, url, entry.get("summary", ""))` if html_bytes, else build partial dict from RSS summary
  - Parse published date: use feedparser entry's `published_parsed` first (via `calendar.timegm`), fall back to `parse_article_date(extracted["published_at"])`, fall back to `datetime.now(timezone.utc)`
  - Build article_data dict: `url`, `url_hash`, `title` (from entry), `body` (from extracted), `body_html` (from extracted), `author` (from extracted), `published_at` (parsed datetime), `lead_image_url` (from extracted), `is_partial` (from extracted), `source_id` (from source)
  - Save: `await save_article(session, article_data)` — increment inserted if True
  - Sleep `settings.article_fetch_delay` seconds between article fetches (politeness) — `await asyncio.sleep(settings.article_fetch_delay)`
- Log `source_complete` at INFO with source name and stats
- Return stats

**`_log_source_error(source_name, error)` and `_reset_failure_count(source_name)`:**
- Increment/reset `_failure_counts[source_name]`
- After `settings.consecutive_failure_threshold` consecutive failures, log at ERROR level
- Below threshold, log at WARNING level

**Create `factfeed/ingestion/scheduler.py`:**

**`create_scheduler(ingestion_fn: Callable) -> AsyncIOScheduler`:**
- Import `AsyncIOScheduler` from `apscheduler.schedulers.asyncio`
- Create scheduler: `AsyncIOScheduler(timezone="UTC")`
- Add job:
  ```python
  scheduler.add_job(
      ingestion_fn,
      trigger="interval",
      minutes=settings.ingest_interval_minutes,
      next_run_time=datetime.now(timezone.utc),  # Run immediately on start
      max_instances=1,        # Single-worker guard
      coalesce=True,          # If missed, run once not multiple
      id="ingestion_cycle",
      name="Ingest RSS feeds",
  )
  ```
- Return the scheduler (caller starts and stops it)
- Import settings from `factfeed.config`

**Create `factfeed/web/main.py`:**

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
import httpx
from factfeed.config import settings
from factfeed.db.session import AsyncSessionLocal
from factfeed.ingestion.scheduler import create_scheduler
from factfeed.ingestion.runner import run_ingestion_cycle
from factfeed.ingestion.persister import seed_sources
from factfeed.ingestion.sources import SOURCES
from factfeed.ingestion.logging import configure_logging

@asynccontextmanager
async def lifespan(app: FastAPI):
    configure_logging()

    # Seed sources into database
    async with AsyncSessionLocal() as session:
        await seed_sources(session, SOURCES)

    # Create shared HTTP client
    async with httpx.AsyncClient(
        headers={"User-Agent": settings.user_agent},
        timeout=httpx.Timeout(connect=5.0, read=30.0, write=5.0, pool=5.0),
        follow_redirects=True,
    ) as http_client:
        # Create and start scheduler
        async def ingestion_job():
            await run_ingestion_cycle(AsyncSessionLocal, http_client)

        scheduler = create_scheduler(ingestion_job)
        scheduler.start()
        yield
        scheduler.shutdown(wait=False)

app = FastAPI(title="FactFeed", lifespan=lifespan)

@app.get("/health")
async def health():
    return {"status": "ok"}
```

Key decisions per CONTEXT.md:
- httpx.Timeout: connect=5s (fail fast on unreachable), read=30s (generous for slow news sites per Pitfall 5 in research)
- `max_instances=1` is the APScheduler single-worker guard (per user decision)
- `next_run_time=datetime.now(timezone.utc)` triggers immediate first run (per user decision)
- `coalesce=True` prevents catch-up flood if scheduler falls behind
- Source seeding happens before scheduler starts (per research Pattern 10)
- A `/health` endpoint is added for basic liveness checking
