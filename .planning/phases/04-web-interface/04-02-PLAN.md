<task type="auto">
  <name>Task 1: Implement composable search query with FTS, filters, and fact-density sort</name>
  <files>factfeed/web/routes/search.py</files>
  <action>
    Replace the stub search route with full implementation:

    1. Create a `SearchParams` Pydantic model (or use individual query params):
       - `q: str = ""` — search keyword
       - `source: int | None = None` — source_id filter
       - `from_: Literal["24h", "7d", "30d"] | None = None` — date range filter (use alias "from" for query param)
       - `sort: Literal["facts", "recency"] = "facts"` — sort order

    2. Create `async def search_articles(session: AsyncSession, params) -> list[Article]`:
       - Base: `select(Article).options(selectinload(Article.source))`
       - If `q` is non-empty: `func.plainto_tsquery("english", q)` with `Article.search_vector.bool_op("@@")(tsquery)`. Use `plainto_tsquery` (safe for user input, treats words as AND).
       - If `source` is set: `.where(Article.source_id == source)`
       - If `from_` is set: compute `cutoff = datetime.utcnow() - timedelta(days={"24h": 1, "7d": 7, "30d": 30}[from_])`, add `.where(Article.published_at >= cutoff)`
       - Sort logic:
         - `"recency"`: `.order_by(Article.published_at.desc())`
         - `"facts"` (default): Build fact-density subquery:
           ```python
           fact_ratio_sq = (
               select(
                   Sentence.article_id,
                   (func.count(Sentence.id).filter(Sentence.label == "fact").cast(Float)
                    / func.nullif(func.count(Sentence.id), 0)
                   ).label("fact_ratio"),
               ).group_by(Sentence.article_id).subquery()
           )
           stmt = stmt.outerjoin(fact_ratio_sq, Article.id == fact_ratio_sq.c.article_id)
                      .order_by(fact_ratio_sq.c.fact_ratio.desc().nullslast())
           ```
       - `.limit(50)`
       - Execute and return `result.scalars().all()`

    3. Update the `GET /` route to:
       - Query all Source rows for the filter dropdown
       - Call `search_articles` with default params (empty query = show all, sorted by facts)
       - Pass results, sources, and params to `search.html`

    4. Update the `GET /search` route to:
       - Accept all query params (q, source, from_, sort)
       - Call `search_articles` with params
       - Check `request.headers.get("HX-Request")`: if true, return `_results.html` partial; if false, return full `search.html`
       - Pass `results`, `sources`, `params` (q, source, from_, sort) to template for form state preservation
       - Use `hx-push-url="true"` in template so URL reflects current filter state (bookmarkable)

    IMPORTANT: Use `selectinload(Article.source)` to avoid lazy-load errors in Jinja2 templates (Pitfall 2 from research). Never use raw string interpolation in FTS queries. Use `.nullslast()` for fact_ratio sort to handle unclassified articles (Pitfall 1).
  </action>
