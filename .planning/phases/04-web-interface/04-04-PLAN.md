    Create `tests/test_web_routes.py` with tests using `httpx.AsyncClient` with `ASGITransport(app=app)` (or FastAPI's `TestClient` if sync is easier). Tests need a database with seeded test data.

    **Test fixtures:**
    - Use the existing `db_session` fixture from conftest (or create a lightweight one).
    - Seed test data: 1 Source, 2 Articles (one with sentences classified, one without), ~5 Sentences on the classified article with mixed labels (2 fact, 1 opinion, 1 mixed, 1 unclear).
    - If full DB fixtures are too heavy, mock the DB session dependency (`app.dependency_overrides[get_db]`) to return a mock session with pre-built Article/Sentence objects.

    **Test cases (minimum):**

    1. `test_homepage_returns_200` — GET `/` returns 200 with HTML content-type, response contains "FactFeed" and "Search"
    2. `test_search_returns_results` — GET `/search?q=test` returns 200, response HTML contains article title from seeded data (if DB available) or just validates structure
    3. `test_search_htmx_returns_partial` — GET `/search?q=test` with `HX-Request: true` header returns HTML that does NOT contain `<!DOCTYPE html>` (partial, not full page)
    4. `test_search_source_filter` — GET `/search?source={source_id}` returns only articles from that source
    5. `test_search_date_filter` — GET `/search?from=7d` returns only articles published within last 7 days
    6. `test_search_sort_facts` — GET `/search?sort=facts` returns 200 (validates the fact-density subquery doesn't error)
    7. `test_search_sort_recency` — GET `/search?sort=recency` returns 200
    8. `test_article_detail_returns_200` — GET `/article/{id}` for existing article returns 200 with article title in HTML
    9. `test_article_detail_404` — GET `/article/99999` returns 404
    10. `test_article_has_sentence_highlighting` — GET `/article/{id}` for classified article, response HTML contains `class="sentence fact"` and `class="sentence opinion"`
    11. `test_article_has_confidence_tooltip` — Response HTML contains `title="High confidence"` or `title="Medium confidence"` or `title="Low confidence"` — does NOT contain raw decimal like `title="0.`
    12. `test_article_has_collapsible_opinions` — Response HTML contains `<details class="opinion-section">` and `<summary`
    13. `test_no_auth_headers` — No `Set-Cookie` header in any response, no `WWW-Authenticate` header
    14. `test_static_css_served` — GET `/static/style.css` returns 200 with CSS content containing `.fact`

    **Approach:** If database-dependent tests are not feasible in the test environment (no PostgreSQL), use `app.dependency_overrides[get_db]` to inject a mock session that returns pre-built ORM-like objects. Use `unittest.mock.AsyncMock` for the session.

    Mark DB-dependent tests with `@pytest.mark.integration` so they can be run separately.
