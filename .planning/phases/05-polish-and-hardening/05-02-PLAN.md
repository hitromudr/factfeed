  <action>
    1. Create `tests/uat/__init__.py` (empty file).

    2. Create `tests/uat/test_uat_articles.py` — a pytest-compatible UAT script that verifies the full user experience on real database content. Mark ALL tests with `@pytest.mark.uat`.

    The UAT script structure:

    **Article selection fixture** (`uat_articles`):
    - Use `AsyncSessionLocal` from `factfeed.db.session` to query the live database (NOT the test db_session fixture — this runs against real ingested data)
    - Query for articles that have BOTH fact AND opinion sentences (to test all highlighting and collapsibility features)
    - Require at least 3 different source IDs represented among the 10 articles (per CONTEXT.md)
    - If fewer than 10 qualifying articles exist, `pytest.skip("Fewer than 10 mixed articles available — run ingestion + NLP classification first")`
    - Return the 10 selected articles (eagerly load sentences and source relationships)

    **Test client fixture** (`uat_client`):
    - Create an httpx AsyncClient with ASGITransport wrapping the FastAPI app
    - Override `get_db` to provide sessions from `AsyncSessionLocal` (real DB, not test rollback)
    - This client hits the real application routes against real data

    **Test functions** (each decorated with `@pytest.mark.uat` and `@pytest.mark.asyncio`):

    a. `test_uat_sentences_have_valid_labels(uat_articles)`:
       - For each of the 10 articles, assert all sentences have `label in {"fact", "opinion", "mixed", "unclear"}`
       - Assert all sentences have `0.0 <= confidence <= 1.0`

    b. `test_uat_article_detail_highlighting(uat_client, uat_articles)`:
       - For each article, GET `/article/{id}` and assert HTTP 200
       - Assert HTML contains `class="sentence fact"` or `class="sentence opinion"` (color-coded highlighting is present)
       - Assert "confidence" appears in the HTML (confidence tooltips rendered)

    c. `test_uat_opinion_collapsible(uat_client, uat_articles)`:
       - For each article (all have opinion sentences by fixture design), GET `/article/{id}`
       - Assert HTML contains `<details` and `<summary` and `"Show opinion content"` (collapsible opinion section)
       - Assert at least one opinion sentence text appears in the response

    d. `test_uat_search_finds_articles(uat_client, uat_articles)`:
       - For each article, extract the first significant word from the title (skip common words like "the", "a", "in")
       - GET `/search?q={keyword}` and assert the article title appears in the results
       - If FTS doesn't match (article may not be indexed yet), log a warning but don't fail — some articles may have titles with only stop words

    Important notes:
    - All tests require a real database with ingested + NLP-classified articles. The `@pytest.mark.uat` marker and `addopts` in pyproject.toml (from Plan 05-01) ensure these never run in the default test suite.
    - Run with: `pytest -m uat --override-ini="addopts="` or `pytest tests/uat/ -m uat --override-ini="addopts="`
    - Document the run command in a docstring at the top of the file
