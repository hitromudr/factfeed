Add a post-ingestion classification step. The research recommends running classification as a **separate post-ingestion pass** — query articles where sentences count = 0, then classify in batches. This decouples ingestion failures from classification failures.

**In `factfeed/nlp/pipeline.py`**, add:
```python
async def classify_unprocessed_articles(session_factory, zs_pipeline, calibrator=None, batch_size=10):
    """Classify articles that have no sentence rows yet."""
    from sqlalchemy import select, func
    from factfeed.db.models import Article, Sentence

    async with session_factory() as session:
        # Find articles with no sentences
        subq = (
            select(Sentence.article_id)
            .distinct()
            .correlate(Article)
            .scalar_subquery()
        )
        stmt = select(Article).where(Article.body.isnot(None)).where(Article.id.notin_(subq)).limit(batch_size)
        result = await session.execute(stmt)
        articles = result.scalars().all()

    for article in articles:
        try:
            results = await classify_article_async(
                article.body, zs_pipeline, calibrator,
                source_name=article.source.name if article.source else ""
            )
            async with session_factory() as session:
                await persist_sentences(article.id, results, session)
        except Exception:
            import structlog
            logger = structlog.get_logger()
            logger.error("classification_failed", article_id=article.id, exc_info=True)
            continue  # Skip failed article, continue with rest
```

**In `factfeed/ingestion/runner.py`**, after the ingestion cycle completes, call the classification step. Add a new function or modify the scheduler job to run classification after ingestion:

1. Read the current runner.py to find where the ingestion cycle completes
2. After `run_ingestion_cycle` finishes, call `classify_unprocessed_articles(session_factory, zs_pipeline, calibrator, batch_size=10)`
3. The `zs_pipeline` and `calibrator` should be created once at app startup (in the FastAPI lifespan or scheduler setup) and passed to the classification function
4. Wrap the classification call in try/except so classification failures never crash the ingestion scheduler
5. Log classification results with structlog: number of articles classified, any failures

**Important considerations:**
- Do NOT import the transformer at module level in runner.py — use lazy loading or pass the pipeline as a parameter
- The classification step runs AFTER ingestion completes (not inline with article fetching)
- Use `batch_size=10` to limit memory usage per classification run
- If the model is not available (e.g., in test environments), skip classification gracefully with a warning log

Add a config option in `factfeed/config.py`:
```python
nlp_enabled: bool = True  # Set to False to disable classification in dev/test
nlp_batch_size: int = 10
```
